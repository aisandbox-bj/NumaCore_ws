Platform

How NumaCore is built.

Three layers, one architecture. The parent brand sits over a single client-facing service and a single internal tool suite. Each layer has one job and one accent colour.

Architecture

Parent. Service. Engine.

NumaCore is the platform brand. NumaCore Stride is the retainer service clients buy. NumaCore Lens is the internal tool suite that makes Stride deliverable at scale. The cascade reads top to bottom: the brand frames the offer; the offer is delivered by the engine.

Architecture principles

Four decisions that are not up for debate.

The NumaCore tools are built against a small set of non-negotiable architecture principles. They keep the platform simple, secure, and portable — and they make every feature decision faster, because the constraints are explicit.

  • Single HTML file per tool No build tools, no frameworks, no external dependencies beyond CDN-hosted libraries. Each tool ships in one file. Opens in any modern browser.
  • Hub-and-spoke JSON session model Lens loads the client JSON at session start. Each tool writes only to its own designated slice. Lens saves the final file at session end. The file is the session record.
  • Local file only — offline-first No server. No login. No network required. The advisor works on a laptop, on site, in a planning room with no signal. The tool runs.
  • Tools are never distributed to clients Stride clients receive reports and exports. They never receive Lens. The tools are the delivery engine of the advisory service, not the product sold.
